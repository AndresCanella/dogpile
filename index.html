How should client-side encryption keys be managed?
Context
The architecture requires zero-liability data handling. Keys must never touch the server. AWS KMS is in the stack but the MCP interface means mutations happen client-side. Two patterns are viable: browser-native SubtleCrypto with keys in IndexedDB, or a local key agent process.
Suggestion
Use SubtleCrypto with non-exportable CryptoKey objects stored in IndexedDB. Keys never leave the browser's secure context. AWS KMS handles key wrapping for backup/recovery. No local agent needed for v1.

Core Mechanism
Hooks, not parsing.

State detection via Claude Code hooks

Dogpile uses Claude Code's native hook system. Each agent emits structured JSON events on lifecycle transitions — PreToolUse, PostToolUse, Notification, Stop.

Hooks write to a shared event bus via SQLite. Sub-agent events include the parent session ID so Dogpile nests them under the right parent and sorts them within their group.

Permission interception: PreToolUse fires before every tool call with tool name and parameters. Dogpile surfaces it inline on the parent's queue card. The hook's response controls whether the tool proceeds.

// .claude/hooks.json
{
  "hooks": {
    "PreToolUse": [{
      "command": "dogpile emit pre-tool",
      "timeout": 30000
    }],
    "PostToolUse": [{
      "command": "dogpile emit post-tool"
    }],
    "Notification": [{
      "command": "dogpile emit notify"
    }],
    "Stop": [{
      "command": "dogpile emit stop"
    }]
  }
}

// Sub-agent event includes parent ID
// → nested under parent in queue
// → sorted within parent's group
// → permission card on parent card

Product Strategy
Companion first. Terminal second.

Validate the queue system before building the terminal.

Phase A: Companion

Standalone queue window alongside cmux. No terminal embedding. Talks to cmux via socket API. Validates queue, sub-agent nesting, double-enter, and permission flow through daily use. ~10-14 days.

Phase B: Terminal

Embed libghostty (using cmux source as reference). Add Agent Focus, Master Diff, and Planning views. All queue logic from Phase A carries forward. cmux removed from stack.
